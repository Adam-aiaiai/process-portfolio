Go Runners (C1)

CPT208 Human-Centric Computing | AY2526 | Active Lifestyles Theme

Project Core Position

Go Runners is a human-centric gamified web application designed for casual runners and running event organizers. Our initial goal is to solve the pain points of "boring solo running" and "inefficient event organization" by integrating playful, location-based features into onsite running experiences—aligning with CPT208's C1 (Active Lifestyles) topic requirements.

Initial Ideas & Key Directions

  • Target Users: Primary = amateur runners (onsite running needs); Secondary = small-scale running event organizers.
  • Core Pain Points to Solve: Solo running is boring/lack of motivation; event organizers struggle to track participants/engage runners.
  • Playful Features (Preliminary): ① Location-based running missions (e.g., "complete 1km in Park A to get a badge"); ② Real-time team running rankings (social interaction); ③ Organizer dashboard (quick event creation/participant tracking).
  • Technical Constraints (CPT208): Use free-tier tools (GitHub Pages/Vercel for deployment), pure web-based (HTML/CSS/JS/lightweight frameworks), mobile-first design.
  • Next Steps: Complete user research (interviews/personas), refine feature list, draft low-fi prototype (Figma).
